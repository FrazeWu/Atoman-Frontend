# Markdown 正文与编辑器预览统一设计

- Date: 2026-06-03
- Status: Draft for review
- Scope: Markdown 渲染语义、正文阅读样式、编辑器预览样式、脚注支持

## 1. 背景

Atoman 目前有多处 Markdown 内容入口，包括博客正文、编辑器分栏预览、Feed 详情、论坛主题和回复。它们都承载同一种内容语义，但样式分散在编辑器、文章详情页和局部页面中，导致正文页与预览区容易出现不同的阅读气质。

本次设计把 Markdown 内容样式收束为一个共享 prose 层。页面仍保留自己的布局容器，但 Markdown 标题、段落、引用、代码块、表格、链接、脚注等内容元素应使用同一套视觉语言。

## 2. 目标

- 编辑器分栏预览与文章详情页正文保持一致的内容气质。
- Feed 和论坛中的 Markdown 正文复用同一套 prose 语言。
- Markdown 渲染器支持最小脚注语法，并保证脚注输出仍经过 sanitize。
- 内容样式集中在一个共享 CSS 来源，避免页面各自复制 `.prose-blog` 内容规则。

## 3. 非目标

本次设计不包含：

- 更换 Markdown 渲染库。
- 改造编辑器交互模式、工具栏或协作逻辑。
- 新增富文本编辑器能力。
- 重做博客详情页、Feed 详情页或论坛页的整体页面布局。
- 改动后端 API 契约。

## 4. 产品规则

- Markdown 内容容器统一使用 `a-prose prose-blog` 类名。
- 页面壳层可以保留自己的间距、滚动、分栏和卡片结构。
- 内容人格只能来自共享 prose 层，不能继续由编辑器预览或文章详情页分别定义。
- 编辑器预览区和正式正文允许容器边界不同，但标题、段落、引用、代码块、链接、表格和脚注的视觉语言必须一致。
- 脚注区应呈现为正文末尾的独立注释区。

## 5. 技术规则

- 保留现有 `marked + DOMPurify` 渲染链。
- 脚注支持在现有 `useMarkdownRenderer.ts` 中以最小预处理/后处理实现。
- sanitize 仍是渲染输出进入 DOM 前的最终出口。
- 共享样式文件为 `web/src/assets/markdown-prose.css`。
- 全局入口 `web/src/main.ts` 注册共享 prose 样式。
- `editor.css` 只保留编辑器容器和 CodeMirror 布局样式，不再承载 Markdown 内容规则。

## 6. 验收标准

- 编辑器分栏预览容器存在，并包含 `a-prose`。
- 文章详情页正文容器包含 `a-prose prose-blog`。
- Feed 详情、Feed 文章侧栏、论坛主题正文、论坛回复正文都接入共享 prose 类名。
- 脚注引用、脚注区和返回链接类名稳定。
- 脚注内容中的危险 HTML 不会绕过 sanitize。
- 视觉检查时，编辑器预览与文章详情页正文的标题、引用、代码块、链接、脚注气质一致。
