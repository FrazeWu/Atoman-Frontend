# 播客上传到收听闭环设计

## 目标

重做播客模块的发现、订阅、播放、收藏、稍后听、时间点评论、收听进度和创作管理闭环。播客模块采用“Apple Podcasts 的清晰收听闭环 + 小宇宙的时间点互动”路线：先保证订阅节目、连续收听、继续听和创作管理完整可用，再把评论和数据沉淀到创作者侧。

## 产品原则

- 播客模块以“订阅节目、连续收听、继续听、时间点互动”为核心，不复制视频模块的观看逻辑。
- 频道在底层仍是统一频道；在播客 UI 中呈现为“节目”。
- 订阅底层复用频道订阅关系；播客订阅页只展示该频道下的播客单集。
- 收藏和稍后听分离：收藏用于长期归档，稍后听用于短期队列。
- 播放使用现有全局音频播放器，不在单集详情页保留孤立原生播放器。

## 现状

- 前端已有 `src/views/podcast/PodcastLayout.vue`、`PodcastHomeView.vue`、`PodcastShowView.vue`、`PodcastEpisodeView.vue`、`PodcastEditorView.vue`。
- 播客侧边栏目前只有“播客大厅”和“播客创作”。
- 播客首页已有热度、精选、探索推荐接口调用，但仍偏简单列表。
- 播客详情页目前使用页面内原生 `<audio>`，没有接入全局播放器。
- 播客编辑页已有音频上传、从视频提取音频、频道/合集选择、草稿/发布。
- 后端已有播客单集 CRUD、音频/封面上传、节目 RSS、单集收藏、节目收藏接口。
- `PodcastEpisode` 复用 `Post` 承载标题、正文、状态和可见性，但播客接口尚未完整接入可见性访问控制。
- 现有播放器已有队列、进度、本地恢复、上一首/下一首、单曲循环，并支持外部 RSS 播客转临时播放条目。

## 对象语义

- 频道：节目
- `PodcastEpisode`：单集
- `Collection`：播客合集、专辑或系列
- 订阅：订阅节目，底层是频道订阅
- 收藏：收藏单集、节目或合集
- 稍后听：播客独立队列，不复用 Feed 稍后阅读

## 上传创建

上传使用两步流程：

1. 上传音频
   - 默认上传音频文件。
   - 保留从视频提取音频能力。
   - 上传阶段显示上传或提取进度。
   - 上传完成后从文件名自动填充标题。
   - 尝试读取音频时长；失败时允许继续发布，但在创作中心显示提醒。

2. 填写信息并发布
   - 必填标题、节目频道、合集、音频。
   - shownotes 建议填写，但第一版不作为硬性必填。
   - 可编辑单集封面、季数、集数、可见性、发布状态。
   - 可见性：公开、仅订阅、私有。
   - 默认节目频道、默认合集、默认发布状态和默认可见性从创作设置读取。
   - 发布成功跳转单集详情页。
   - 保存草稿跳转单集管理页。

封面 fallback 顺序：

1. 单集封面
2. 合集封面
3. 节目封面

## 可见性和权限

发布状态和可见性分离：

- 草稿不对外可见。
- 发布后才按公开、仅订阅、私有控制访问。

访问规则：

- 公开单集：所有人可听。
- 仅订阅单集：订阅该节目的用户可听。
- 私有单集：作者和管理员可听。
- 未订阅用户打开仅订阅单集分享链接时，看到订阅提示。
- 私有单集隐藏分享按钮。
- 单集可见性标识只在创作管理显示，普通收听页不显示系统状态。

## 播放器适配

现有 `Song` 类型升级为播放器条目，增加来源字段：

- `source_type`: `music | podcast_episode | feed_podcast`
- `source_id`: 原始实体 ID

规则：

- 音乐播放才调用音乐播放统计。
- 站内播客播放调用播客播放统计和进度同步。
- 外部 RSS 播客不进入站内播客收藏和统计。
- 播客单集通过 `createPodcastEpisodeSong(episode)` 转成播放器条目。
- 播客队列可以来自节目页、订阅页、收藏页、稍后听或详情页右侧列表。

播客播放器条目映射：

- `title`: 单集标题
- `artist`: 节目名
- `album`: 合集名或节目名
- `audio_url`: 单集音频地址
- `cover_url`: 单集封面 fallback 后结果
- `lyrics`: shownotes
- `source_type`: `podcast_episode`
- `source_id`: 单集 ID

播放器 UI：

- 音乐播放时保留歌词、音乐收藏、添加到歌单。
- 播客播放时“词”面板显示“节目说明”，隐藏音乐收藏和添加到歌单，显示收藏单集和稍后听。
- 播客 shownotes 自动解析时间轴，点击时间点跳转播放进度。

## Shownotes 时间轴

解析规则：

- 支持 `00:32`、`12:05`、`01:02:30`。
- 每一行如果包含时间点，就提取为一个可点击章节。
- 点击时间点调用播放器 `seek(seconds)`。
- 解析失败的行按普通 shownotes 显示。
- 不新增章节表；第一版只从 `Post.content` 动态解析。

展示位置：

- 单集详情页 shownotes。
- 全局播放器节目说明面板。

## 收听进度

登录用户使用后端同步进度，未登录用户使用 localStorage 临时保存。

新增进度记录字段：

- 用户
- 单集
- 当前秒数
- 总时长
- 是否听完
- 最后收听时间

完成规则：

- 播放到 95% 以上，或剩余少于 60 秒，标记为听完。
- 用户重新 seek 到前面继续听时，可以改回未听完。

前端上报规则：

- 播客播放开始时拉取该单集进度并恢复。
- 播放过程中节流上报，例如每 15 秒一次。
- 暂停、切换单集、页面卸载前补报一次。
- 同步逻辑放在播放器 store 或播放器进度 composable 中，不依赖详情页组件生命周期。

## 评论

播客使用时间点评论：

- `target_type = podcast_episode`
- `target_id = episode.id`
- 普通评论允许存在，`timestamp_sec = null`
- 时间点评论记录当前播放器秒数

详情页交互：

- 评论区默认展示全部评论。
- 有时间点的评论显示时间标签。
- 点击时间标签跳转到对应播放位置。
- 评论输入框提供“发表评论”和“带当前时间评论”。
- 当前播放器播放的不是本单集时，“带当前时间评论”不可用。

创作端互动管理：

- 全部评论
- 时间点评论
- 点击评论可跳到对应单集和时间点
- 作者可管理自己节目下评论

## 探索页

探索页同时展示节目和单集：

- 新单集
- 热门节目
- 热门单集
- 精选节目或单集

顶部模式仍保留：

- 热度
- 精选
- 探索

首页不显示草稿、私有且无权限访问的单集。

## 订阅页

订阅页是播客更新流：

- 主体只显示已订阅节目下的播客单集。
- 条目采用 Feed 列表形态，突出更新时间、节目来源、时长和收听状态。
- 侧边栏显示已订阅节目，用于筛选更新。
- 收藏合集的新单集也可以进入更新流，但需要明确标记来源。

## 收藏页

收藏页 tab：

1. 单集
2. 节目
3. 合集
4. 稍后听

默认进入单集 tab。

稍后听规则：

- 播客独立队列，不复用 Feed 稍后阅读。
- 在稍后听 tab 点播放时，把整个稍后听列表设为播放器队列。
- 播完不自动移除，第一版保持简单。
