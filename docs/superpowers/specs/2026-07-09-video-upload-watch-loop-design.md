# 视频上传到观看闭环设计

## 目标

重做视频模块的上传、创作管理、观看、订阅、收藏和数据闭环。用户可以从本地上传或外链创建视频，创作者能管理内容状态和数据，观看者能在详情页完成播放、订阅、收藏、稍后看和分享。

## 现状

- 前端已有 `src/views/video/VideoHomeView.vue`、`VideoEditorView.vue`、`VideoManageView.vue`、`VideoDetailView.vue`、`VideoSubscriptionsView.vue`。
- 视频首页已有热度、精选、探索推荐接口调用，但页面仍带“推荐区”概念。
- 视频订阅页目前是空壳。
- 视频卡片已有“稍后看”视觉入口，但不是完整动作。
- 详情页已有本地播放、外链嵌入、播放进度、时间戳评论、推荐视频和互动条。
- 后端已有 `/api/v1/videos`、上传视频/封面、视频书签、推荐、评论、点赞、处理任务和 Swagger 文档。
- `model.Video` 已有 `status`、`visibility`、`processing_status`、`processing_error`、`preview_thumbnails`、`view_count`、`like_count`、`comment_count`、`Collections`。
- 当前抽帧/预览图链路存在不可用问题，本次作为 bug 修复。

## 上传创建

上传使用两步流程，参考音乐创建专辑的分步体验：

1. 上传来源
   - 默认本地上传。
   - 可切换外链视频。
   - 本地上传阶段显示上传进度。
   - 外链支持 YouTube、Bilibili 内嵌；其他链接作为外部链接。
   - 粘贴外链后尝试解析标题和封面；失败时允许手动填写。

2. 填写信息并发布
   - 必填标题、频道、合集。
   - 默认选中默认频道和默认合集。
   - 必须有频道，也必须有合集。
   - 可编辑简介、封面、标签、可见性、发布状态。
   - 可见性：公开、仅订阅、私有。
   - 默认可见性从创作设置读取，初始为公开。
   - 发布状态：默认发布，可切换保存草稿。
   - 发布成功跳转视频详情页。
   - 保存草稿跳转创作管理页。

封面逻辑：

- 本地视频尝试抽帧生成封面。
- 抽帧失败时显示失败状态，允许重试抽帧或手动上传封面。
- 抽帧不可用是本次必须修复的 bug。

处理状态：

- 上传表单只显示本地文件上传进度。
- 内容管理显示视频文件处理状态，不显示百分比。
- 内容管理状态至少包括：可观看、转码中、处理失败。
- 处理失败可执行：编辑信息、重新上传文件、重新处理、删除。

## 可见性和权限

发布状态和可见性分离：

- 草稿不对外可见。
- 发布后才按公开、仅订阅、私有控制访问。

访问规则：

- 公开视频：所有人可看。
- 仅订阅视频：订阅频道的人可看。
- 私有视频：作者和管理员可看。
- 未订阅用户打开仅订阅视频分享链接时，看到订阅提示。
- 私有视频隐藏分享按钮。
- 视频可见性标识只在创作管理显示，普通观看页不显示系统状态。

频道和合集语义：

- 频道动作叫订阅。
- 合集动作叫收藏。
- 收藏合集后，合集新视频也进入订阅更新流。

## 创作中心

### Dashboard

Dashboard 是创作者中心首页，包含：

- 上传入口。
- 最近视频。
- 内容管理入口。
- 关键数据卡片：播放、评论、点赞、收藏、分享、订阅数。
- 订阅数概览。
- 收益占位：只显示暂无收益数据，不接收益业务。
- 问题提醒：上传/处理失败、抽帧失败、草稿未发布、外链不可播放、缺封面等可真实判断的问题。

不做活动/公告模块。

### 内容管理

内容管理采用视频后台式列表。字段：

- 封面/标题
- 状态
- 频道
- 合集
- 播放
- 评论
- 点赞
- 收藏
- 分享
- 发布时间
- 操作

筛选：

- 全部
- 公开
- 仅订阅
- 私有
- 草稿
- 处理失败

搜索范围：

- 标题
- 简介
- 频道
- 合集

排序：

- 默认按更新时间倒序。

操作：

- 编辑
- 删除，二次确认后执行。
- 分享。私有视频隐藏分享入口。
- 重新上传文件。
- 重新处理。

移动频道/合集只在编辑视频时处理，不在列表行内做快速移动。

### 数据中心

数据中心包含：

- 总览卡片。
- 视频排行。
- 趋势图。
- 时间范围：7 天、28 天、90 天可切换，默认 28 天。

### 互动管理

互动管理包含两个 tab：

- 全部评论
- 时间戳评论

时间戳评论继续复用详情页已有的时间戳评论模型和提取逻辑。

### 创作设置

创作设置包含：

- 默认频道。
- 默认合集。
- 默认发布/草稿。
- 默认可见性：公开、仅订阅、私有，初始为公开。
- 是否连续播放。

连续播放也要接入现有全局设置页；默认关闭。

## 观看详情页

布局：

- 左侧主播放器和视频信息。
- 右侧先显示当前合集，再显示推荐视频。
- 右侧合集列表高度和播放器高度一致，参考 Bilibili 详情页；合集内滚动，高亮当前视频。

播放：

- 默认不连续播放。
- 用户开启连续播放后，优先在同一合集内播放下一条；合集结束后再进入推荐视频。
- 观看进度保存在本地浏览器；本次不做跨设备同步。

信息区：

- 标题下方显示频道入口和频道订阅按钮。
- 右侧合集区显示合集收藏按钮。
- 简介默认折叠两三行，点击展开。

操作顺序：

1. 点赞
2. 收藏/稍后看
3. 分享
4. 更多

更多菜单：

- 举报
- 复制链接
- 作者管理入口

分享：

- 使用 `navigator.share`。
- 不支持系统分享时复制链接兜底。
- 系统分享成功或复制链接成功后才统计分享次数。
- 分享入口放在详情页和创作管理列表，不放首页视频卡片。

评论：

- 桌面默认展开。
- 移动端默认折叠。

## 探索首页

首页不做顶部推荐区。

首页采用类似订阅模块的切换：

- 热度：播放/互动排序。
- 精选：人工或推荐接口。
- 探索：最新或随机。

首页只展示已发布且当前用户有权限访问的视频，不显示草稿。

## 订阅页

订阅页是视频更新流：

- 主体只显示视频更新。
- 条目采用 Feed 列表形态，突出更新时间和来源。
- 显示订阅频道的公开和仅订阅视频。
- 显示收藏合集的新视频。
- 侧边栏显示订阅频道和收藏合集，用于筛选更新。

## 收藏页

收藏页 tab：

1. 视频
2. 频道
3. 合集
4. 稍后看

默认进入视频 tab。

稍后看是收藏的一种筛选，不单独做平行系统。

频道和合集详情页也要提供收藏/订阅状态入口。

## 频道和合集详情页

频道详情页：

- 频道信息。
- 订阅按钮。
- 视频列表。
- 作者可见“管理”按钮，进入创作中心并筛选该频道。

合集详情页：

- 合集信息。
- 收藏按钮。
- 视频列表。
- 连续播放开关。
- 作者可见“管理”按钮，进入创作中心并筛选该合集。

## 后端接口原则

优先扩展现有视频接口，而不是新建平行模块。

需要后端支持：

- `visibility` 明确为 `public`、`subscribers`、`private`。现有 `followers` 语义要迁移或兼容。
- 创建/更新视频时校验视频频道和合集必填，且都属于当前用户的视频内容空间。
- 列表接口支持创作者管理参数：状态、可见性、搜索、频道、合集、更新时间排序。
- 详情接口按访问规则返回视频，支持公开视频、仅订阅视频、私有视频。
- 频道订阅状态与合集收藏状态查询。
- 频道订阅和合集收藏驱动订阅页更新流。
- 视频、频道、合集、稍后看收藏接口。
- 分享成功统计接口。
- 处理任务重试接口。
- 本地抽帧/预览图生成修复。
- 数据中心聚合接口。
- 互动管理评论列表接口，支持时间戳评论筛选。

Swagger 文档必须同步更新。

## 前端实现原则

- 复用现有 UI 组件和音乐模块的抽屉/分步创建模式。
- 复用 Feed/订阅模块的侧边栏、列表、订阅更新流思路。
- 不在普通用户界面暴露研发状态或内部词汇。
- 用户文案保持短句，错误只告诉用户如何修正。
- 首页、订阅页、详情页是观众视角；创作中心是作者视角。
- 不把分享入口放进首页视频卡片。
