# Favicon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show the Atoman image from `public` as the browser tab favicon.

**Architecture:** Vite serves files from `public/` at the site root. The implementation creates the conventional `favicon.png` static asset from the existing `favio.png` image and points `index.html` at `/favicon.png`.

**Tech Stack:** Vue 3, Vite, static HTML metadata, `public/` assets.

---

### Task 1: Static Favicon Asset And HTML Link

**Files:**
- Create: `public/favicon.png`
- Modify: `index.html`
- Verify: `bun run build`

- [ ] **Step 1: Copy the existing image to the conventional filename**

Run:

```bash
cp public/favio.png public/favicon.png
```

Expected: `public/favicon.png` exists and contains the same image content as `public/favio.png`.

- [ ] **Step 2: Update the favicon link**

In `index.html`, replace the existing favicon line with:

```html
    <link rel="icon" type="image/png" href="/favicon.png" />
```

The resulting `<head>` should remain minimal and valid:

```html
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Atoman</title>
    <link rel="icon" type="image/png" href="/favicon.png" />
  </head>
```

- [ ] **Step 3: Build the frontend**

Run:

```bash
bun run build
```

Expected: Vite completes successfully and includes the static public asset in `dist/favicon.png`.

- [ ] **Step 4: Commit only favicon-related files**

Run:

```bash
git add docs/specs/2026-06-17-favicon-design.md docs/superpowers/plans/2026-06-17-favicon.md index.html public/favicon.png
git commit -m "feat: add site favicon"
```

Expected: The commit includes only the favicon spec, plan, HTML metadata update, and new static favicon asset.
