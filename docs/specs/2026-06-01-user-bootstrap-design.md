# 新用户初始化设计

## 一、目标

新用户注册成功后，系统自动创建一组最小可用的默认对象，保证用户进入站点后立即具备基础发布能力与基础收藏能力，不依赖后续页面懒初始化。

本次只覆盖注册成功后的初始化骨架，不扩展到推荐内容、官方引导、稍后读或其他非必需容器。

## 二、确认后的范围

注册成功后，系统必须确保以下对象存在：

1. `UserSettings`
2. 同名默认频道
3. 该频道下的默认合集
4. 一条“订阅自己”的默认订阅
5. 一个名为`默认收藏`的书签文件夹

其中：

1. `默认收藏`只是系统预置的第一个书签文件夹
2. 用户后续可以自行创建更多书签文件夹
3. 本次不创建“稍后读”
4. 本次不创建“官方订阅”
5. 本次不创建额外默认频道或额外默认合集

## 三、现状与问题

当前注册流程已经在事务内完成以下动作：

1. 创建 `User`
2. 创建 `UserSettings`
3. 创建默认频道
4. 创建默认合集

但当前实现存在两个问题：

1. “订阅自己”还没有被定义成统一的新用户初始化能力
2. 现有 `blog` 模块服务会在创建默认频道时自动执行“订阅自己的频道”，其语义是 `internal_channel`，与本次确认的“订阅自己用户”不一致

另外，默认书签文件夹目前也没有在注册时创建。

## 四、产品语义

### 4.1 订阅自己

“订阅自己”应表达“关注这个用户的站内输出”，而不是“关注这个用户当前的某一个频道”。

因此默认订阅的目标类型应为：

- `internal_user`

而不是：

- `internal_channel`

这样做的原因：

1. 语义更稳定，用户新增频道后不需要改变默认订阅定义
2. 与现有订阅系统支持的 `internal_user` 类型一致
3. 避免把用户级关注错误绑定到默认频道这一实现细节上

### 4.2 默认收藏

默认收藏对应现有书签文件夹能力，即：

- `bookmark_folders`

默认名称固定为：

- `默认收藏`

这个对象只是初始容器，不限制用户后续新建更多收藏文件夹。

## 五、设计原则

### 5.1 注册时一次性初始化

所有默认对象都在注册成功的同一条初始化链路中完成，避免部分注册时创建、部分首次访问页面时懒创建的分裂状态。

### 5.2 使用 ensure 语义

初始化逻辑必须可重复执行，不因为重复调用而报错，也不因为重复调用而创建多份默认数据。

这意味着每个默认对象都应遵循：

1. 已存在则直接复用
2. 缺失则补建
3. 不做重复插入

### 5.3 最小改动

只解决本次确认的 5 个默认对象，不引入新的抽象层级，不顺手扩展到其他模块。

## 六、推荐架构

新增一个统一的新用户初始化入口，负责串起所有默认对象的 ensure 行为。

推荐职责分布：

1. `auth_handler` 负责注册请求、用户创建、事务组织
2. 新用户初始化服务负责默认对象补齐
3. 现有 blog 默认频道/默认合集逻辑继续复用
4. 订阅与默认收藏初始化从散落逻辑收敛到统一入口

推荐初始化顺序：

1. 创建 `User`
2. 创建 `UserSettings`
3. ensure 默认频道
4. ensure 默认合集
5. ensure 默认订阅分组
6. ensure 一条 `internal_user` 的“订阅自己”
7. ensure 一个名为`默认收藏`的书签文件夹

## 七、实现边界

### 7.1 默认频道与默认合集

继续复用现有逻辑，不重写模型，不改默认合集的命名策略。

### 7.2 订阅逻辑

需要把“新用户自动订阅”的定义从“订阅自己的默认频道”调整为“订阅自己这个用户”。

如果现有默认频道创建逻辑内置了自动订阅频道行为，后续实现必须处理这个耦合，避免注册后同时出现：

1. 一条 `internal_channel`
2. 一条 `internal_user`

本次目标是注册后只保留我们确认过的默认订阅语义，即：

1. 至少有且仅有一条符合初始化语义的“订阅自己”
2. 该订阅的目标类型为 `internal_user`

### 7.3 默认收藏

默认收藏使用现有 `BookmarkFolder` 模型即可，不新增字段，不增加“系统文件夹”标记。

本次只要求名称固定、自动创建、可重复执行不重复。

## 八、错误处理

初始化属于注册事务的一部分。

约束如下：

1. 任一默认对象初始化失败，则整个注册失败
2. 不接受“用户已创建但默认骨架不完整”的部分成功状态
3. 错误信息保持现有后端风格，使用简单直接的失败返回即可

## 九、测试策略

本次只做最小必要测试，重点验证注册后的最终状态。

至少覆盖：

1. 注册成功后会创建默认频道
2. 注册成功后会创建默认合集
3. 注册成功后会创建默认订阅分组
4. 注册成功后会创建一条 `internal_user` 的订阅自己记录
5. 注册成功后会创建名为`默认收藏`的书签文件夹
6. 重复执行初始化不会产生重复的默认订阅或默认收藏

## 十、不做的事

本次明确不做：

1. 稍后读初始化
2. 官方账号或官方频道默认订阅
3. 多个默认收藏夹
4. 默认草稿、默认播放列表、默认论坛分组
5. 新的 API 接口
6. 数据库结构变更

## 十一、结论

本次新用户初始化以“最小但完整”为原则，注册成功后统一确保以下骨架存在：

1. `UserSettings`
2. 同名默认频道
3. 默认合集
4. `internal_user` 语义的订阅自己
5. 名为`默认收藏`的书签文件夹

后续实现应优先收敛现有“默认频道创建时自动订阅频道”的耦合，让注册初始化语义回到用户级，而不是频道级。
