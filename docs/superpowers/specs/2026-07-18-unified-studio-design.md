# 统一创作中心设计

## 目标

将现有以视频创作中心为基础的规划扩展为统一 Studio，集中管理站内博客、播客和视频创作。用户从全站顶部栏的唯一入口进入 Studio，在同一个当前频道下查看三个模块的 Dashboard 摘要，并分别管理内容、数据、互动和创作设置。

## 已确认决策

- 全站只保留一个“创作中心”按钮，位于登录后的顶部栏，跳转 `/studio`。
- 删除博客、播客、视频模块原有的创作入口，不保留旧入口和旧 API 兼容。
- Studio 顶部保存一个全局当前频道状态，博客、播客和视频共用该频道。
- 一个频道可以同时包含博客、播客和视频。
- Dashboard 只有一页，按博客、播客、视频展示三个独立摘要区，指标不跨模块相加。
- 每个摘要区只展示关键数据、最近内容、问题提醒、新建入口和管理入口。
- Studio 侧边栏包含 Dashboard、博客、播客、视频和频道设置。
- 每个内容模块包含内容管理、数据中心、互动管理和创作设置。
- 不提供独立合集页。内容管理包含合集筛选，筛选旁的管理按钮打开合集侧边面板。
- 当前合集筛选会传给编辑器；从“全部合集”创建时，用户可在编辑器内选择合集。
- 本次包含前端、后端、数据库迁移和 API 文档，不包含 iOS。
- 音乐、论坛、时间线及其他模块不进入 Studio。

## 全站边界

- 博客、播客、视频是用户在站内创作、发布和被关注的内容。
- 关注用户表示接收其名下所有频道的更新。
- 订阅频道表示接收该频道内三个模块及所有合集的更新。
- 订阅模块继续聚合站内博客、播客、视频更新以及外部 RSS。
- 音乐保持 Wiki 式上传、编辑和站内订阅，不进入统一订阅流和 Studio。
- 论坛、时间线及其他模块保持独立。

## Studio 壳层

Studio 使用独立布局，不复用博客、播客或视频的消费侧边栏。

桌面端：

- 顶部区域显示当前频道选择器和频道管理入口。
- 左侧显示 Studio 一级导航。
- 右侧显示当前页面。
- 频道切换不改变当前模块和子页面，只重新加载频道范围内的数据。

移动端：

- 当前频道仍固定在 Studio 页头。
- 左侧导航收进菜单面板。
- 内容筛选允许换行或进入筛选面板，不能压缩内容标题和操作按钮。

如果用户还没有频道，Studio 显示创建频道空状态。创建成功后自动设为当前频道，并进入 Dashboard。

## 频道模型

### 统一语义

`Channel` 不再携带 `blog | podcast | video` 内容类型，也不再存在每个模块各自的默认频道。频道只表示一个创作者发布空间，三个模块的内容都通过 `channel_id` 归属同一个频道。

当前频道是用户级 Studio 状态：

```text
UserStudioState
├── user_id，唯一
└── channel_id，可空
```

状态保存在后端，以便不同浏览器保持一致。删除当前频道后，后端选择该用户最早创建的剩余频道；没有剩余频道时返回空状态。

新用户注册时沿用现有自动初始化体验，但只创建一个以用户名命名的频道，将它设为当前频道，并为博客、播客、视频分别创建一个默认合集，不再创建三套默认频道。

频道设置提供：

- 创建频道。
- 修改名称、简介和封面。
- 切换当前频道。
- 删除空频道；存在内容时返回冲突，不执行级联删除。

### 现有数据迁移

- 每个旧频道原样保留为一个新的全局频道，不按名称自动合并。
- 旧频道内的文章、播客单集、视频和订阅关系保持原 `channel_id`。
- 每个用户的初始当前频道按旧默认频道优先级选择：博客、播客、视频；均不存在时选择最早创建的频道。
- 完成合集类型回填后，移除频道 `content_type`、`is_default` 和按模块保存的 `user_default_channels`。
- 迁移不删除内容，也不尝试猜测不同名称的旧频道是否属于同一个品牌。

## 合集模型

合集属于当前频道，同时必须明确所属模块：

```text
Collection
├── channel_id
├── content_type: blog | podcast | video
├── name
├── description
├── cover_url
└── is_default
```

约束：

- 同一频道、同一模块内合集名称唯一。
- 每个频道、每个模块最多一个默认合集。
- 博客内容只能关联博客合集，播客单集只能关联播客合集，视频只能关联视频合集。
- 草稿允许暂时没有合集；发布前至少选择一个合集。
- 删除合集只删除关联关系，内容继续保留；如果它是创作设置中的默认合集，同时清空该设置。

迁移时根据旧频道 `content_type` 回填现有合集的 `content_type`，再移除频道类型字段，因此现有合集不会丢失。

## Dashboard

Dashboard 同页展示三个全宽摘要区，不使用三套独立 Dashboard 路由。

公共内容：

- 模块名称。
- 新建内容按钮。
- 进入内容管理按钮。
- 最近三条内容。
- 可真实判断的问题提醒。

指标：

| 模块 | 指标 |
| --- | --- |
| 博客 | 阅读、评论、点赞、收藏、分享 |
| 播客 | 播放、完播、评论、收藏、分享 |
| 视频 | 播放、评论、点赞、收藏、分享 |

频道订阅数属于全局频道，只在 Dashboard 页头显示一次，不复制到三个模块，也不与其他指标相加。

问题提醒：

- 博客：草稿、缺封面、未选合集。
- 播客：草稿、缺音频、缺封面、未选合集、待回复评论。
- 视频：草稿、缺封面、未选合集、处理失败、外链不可播放、待回复评论。

每个模块加载和报错相互隔离。一个模块失败时显示该区域的重试状态，不阻塞另外两个区域。

## 内容管理

三个模块共享页面骨架和筛选协议，但使用各自的列表字段和操作。

公共筛选：

- 搜索标题和简介。
- 状态：全部、已发布、草稿。
- 可见性：全部、公开、仅订阅、私有。
- 合集：全部合集或指定合集。
- 排序：默认按更新时间倒序。

公共操作：

- 新建。
- 编辑。
- 发布或转为草稿。
- 删除，执行前二次确认。
- 公开和仅订阅内容可分享；私有内容不显示分享入口。

模块字段：

| 模块 | 额外字段和操作 |
| --- | --- |
| 博客 | 阅读、评论、点赞、收藏、分享、发布时间 |
| 播客 | 时长、播放、完播、评论、收藏、分享、音频重新上传 |
| 视频 | 时长、播放、评论、点赞、收藏、分享、处理状态、文件重新上传、重新处理 |

### 合集筛选与管理

- 内容管理顶部的合集选择器负责筛选。
- 选择器旁使用管理图标按钮，悬停提示“管理合集”。
- 点击后打开侧边面板，不进入新页面。
- 面板支持新建、重命名、修改描述和删除合集。
- 从指定合集筛选状态点击新建，编辑器自动带入该合集。
- 从“全部合集”点击新建，编辑器不预选普通合集，用户可在写作过程中选择。

## 数据中心

每个模块独立展示：

- 总览指标。
- 内容排行。
- 趋势图。
- 7 天、28 天、90 天范围，默认 28 天。

阅读、播放、完播和分享需要按事件时间统计。点赞、评论和收藏直接使用现有带时间戳的互动记录。后端新增最小的内容指标事件记录，只承载现有模型无法按时间还原的阅读、播放、完播和分享，不建立第二套内容模型。

## 互动管理

互动管理复用统一评论核心，并按当前频道和模块筛选创作者自己内容下的评论。

- 默认按最新评论排序。
- 支持未回复筛选。
- 支持从评论跳转到对应内容。
- 视频保留“全部评论”和“时间戳评论”筛选。
- 播客保留带时间锚点的评论信息。
- 回复、删除和置顶继续使用统一评论接口，不复制评论业务逻辑。

## 创作设置

设置以“用户 + 频道 + 模块”为范围：

```text
StudioModuleSettings
├── user_id
├── channel_id
├── content_type
├── default_collection_id，可空
├── default_visibility
├── default_publish_status
└── autoplay_enabled
```

- 默认可见性：公开、仅订阅、私有；初始为公开。
- 默认发布状态：已发布或草稿；初始为已发布。
- 默认合集必须属于当前频道和当前模块。
- 连续播放只用于播客和视频，博客不显示该控件。

切换频道后读取该频道自己的模块设置，不跨频道复用默认合集。

## 编辑器流程

- 编辑器从 Studio 状态读取当前频道，不再显示频道选择器。
- 编辑既有内容时，以内容自己的频道为准；如果不是当前频道，Studio 先切换到该频道再打开编辑器。
- 路由查询参数 `collection` 只用于预选合集，最终以保存请求中的 `collection_ids` 为准。
- 草稿可无合集保存；发布请求若没有合集返回 `400`。
- 发布成功进入内容详情；保存草稿返回对应模块的内容管理页，并保留合集筛选。

## API

Studio API 统一挂在 `/api/v1/studio`，全部要求登录：

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| GET | `/state` | 当前频道和用户频道列表 |
| PATCH | `/state` | 切换当前频道 |
| GET/POST | `/channels` | 列出或创建频道 |
| PATCH/DELETE | `/channels/:id` | 修改或删除频道 |
| GET | `/dashboard` | 三模块摘要和频道订阅数 |
| GET | `/:module/contents` | 内容管理列表与筛选 |
| GET/POST | `/:module/collections` | 合集列表或新建合集 |
| PATCH/DELETE | `/:module/collections/:id` | 修改或删除合集 |
| GET | `/:module/analytics` | 模块数据中心 |
| GET | `/:module/interactions` | 模块互动列表 |
| GET/PATCH | `/:module/settings` | 模块创作设置 |
| POST | `/:module/contents/:id/share` | 记录分享并返回分享地址 |

`module` 只接受 `blog`、`podcast`、`video`。所有查询默认使用当前频道，也允许显式传入用户拥有的 `channel_id`。服务层必须统一校验频道所有权和合集类型，具体内容查询仍由三个模块各自实现，不在一个巨大处理器里用条件分支复制业务。

消费侧的公开内容、详情和订阅 API 保持各模块职责；只移除旧创作中心、旧默认频道和旧创作者统计 API。

## 权限与错误

- 未登录访问 Studio 返回 `401`，前端进入登录页。
- 没有对应发布权限的模块不显示新建按钮，直接访问写入路由返回 `403`。
- 访问不属于自己的频道、合集或内容返回 `403`。
- 当前频道不存在返回 `404`，前端重新加载 Studio 状态。
- 删除存在内容的频道返回 `409`。
- 合集类型和内容类型不匹配返回 `400`。
- Dashboard 单模块失败返回该模块错误字段；整个请求只有状态和所有模块都无法读取时才失败。

## 前端组件边界

- `StudioLayout`：频道状态、一级导航和响应式壳层。
- `StudioDashboardView`：组合三个摘要区，不包含模块查询细节。
- `StudioDashboardSection`：呈现单个模块摘要。
- `StudioModuleLayout`：模块二级导航。
- `StudioContentView`：公共筛选、分页和内容表格。
- `StudioContentTable`：根据模块列配置呈现行，不执行请求。
- `StudioCollectionSheet`：合集增删改。
- `StudioAnalyticsView`：指标、排行和趋势。
- `StudioInteractionsView`：评论筛选和统一评论操作入口。
- `StudioSettingsView`：模块默认值。
- `StudioChannelView`：频道增删改和当前频道切换。
- `studio` Pinia store：状态、Dashboard 和模块请求；不保存编辑器正文。

## 验收标准

- 登录用户可从顶部栏唯一按钮进入 `/studio`。
- 博客、播客、视频消费模块不再出现创作入口，旧创作路由不可访问。
- 切换当前频道后 Dashboard 和当前模块数据同步切换。
- Dashboard 同页显示三个独立摘要区，一个模块失败不影响另外两个。
- 三个模块的内容管理都支持状态、可见性和合集筛选。
- 合集可在内容管理侧边面板中增删改，没有独立合集页面。
- 从合集筛选创建内容会预选合集，从全部合集创建可在编辑器内选择。
- 编辑器不再要求选择频道，发布内容归属当前频道。
- 频道可同时拥有博客、播客和视频内容，合集只能容纳对应模块内容。
- 现有频道、合集和内容在迁移后仍可访问。
- 后端 `go build ./...`、Studio 相关 Go 测试、前端类型检查和 Studio 相关单元测试通过。

## 不纳入本次

- iOS 改造。
- 音乐、论坛、时间线的创作入口整合。
- 收益、活动和公告。
- 跨频道批量移动内容。
- 自动合并旧频道。
- 恢复旧创作路由或旧创作者 API。
