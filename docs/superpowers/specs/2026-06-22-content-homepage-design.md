# 内容模块首页设计

## 背景

当前内容模块已经有：

- 创作页：面向频道主和创作者的工作台
- 文章页：面向读者的公开文章列表
- 播客页：面向读者的公开播客列表
- 视频页：面向读者的公开视频列表

但模块还没有一个真正独立的“内容首页”。此前根路径直接进入创作路径，不符合现在“读者优先”的内容站定位。新的首页需要优先服务游客和读者，让用户进入内容站后先看到值得点击的内容，而不是先进入工作流。

## 目标

新首页需要满足以下目标：

1. 进入内容模块时，首先服务读者的浏览和发现需求。
2. 第一屏要有明确的内容吸引力，但不靠解释性文字堆叠。
3. 首页结构要清楚区分文章、播客、视频三种内容。
4. 创作入口仍然保留，但必须降级为次要动作。
5. 页面要适合持续刷新最新内容，而不是依赖复杂的首页运营结构。

## 非目标

本次设计不包含以下内容：

1. 不重做创作页的信息架构。
2. 不把文章、播客、视频合并成一条长期混排流。
3. 不引入首页轮播。
4. 不在首屏加入解释性说明、统计数字、品牌口号或长摘要。
5. 不要求后端先提供新的复杂推荐算法，首页可以先建立在现有最新内容能力之上。

## 用户与优先级

首页的第一优先用户是游客和普通读者。

他们的主要目标是：

1. 快速理解这里有哪几类内容。
2. 看到值得点开的内容。
3. 顺着类型继续浏览文章、播客或视频。

创作者和频道主不是首页的第一优先用户，但仍需被照顾：

1. 他们进入内容模块后仍应能快速找到创作入口。
2. 他们不应该被首页阻断，只是不应成为首页主叙事中心。

## 信息架构

首页采用两段式结构：

1. 首屏：精选门面区
2. 第二屏：文章、播客、视频三个内容分区

首页不设置第三屏混合流。

### 首屏

首屏只承担“门面”和“点击动机”两件事，不承担解释站点的任务。

首屏结构：

1. 1 个主精选内容位
2. 2 个次精选内容位
3. 创作入口作为次级动作保留

首屏不放说明文案，不放大段摘要，不放运营口号。

每个精选位只保留必要信息：

1. 标题
2. 内容类型
3. 封面或视觉素材
4. 作者或频道信息

精选内容不预设类型分配。主位和次位都可以来自文章、播客或视频，取决于内容表现力和编辑权重。

### 第二屏

第二屏直接接管首页的主要浏览任务。

内容分区固定为：

1. 文章
2. 播客
3. 视频

每个分区都展示最新内容，不提供复杂排序切换作为首页主行为。

每个分区：

1. 展示 4 到 6 条最新内容
2. 提供一个“查看全部”入口
3. 用清晰的类型视觉语言区分三类内容

## 视觉与交互结构

### 首屏与第二屏的关系

首屏是强门面，但第二屏需要很快接管页面节奏。

这意味着：

1. 首屏不能高到像独立宣传页。
2. 用户进入页面后，应该能较快意识到下面还有明确的内容结构。
3. 页面视觉重点要从“精选内容”自然过渡到“持续浏览”。

首页应避免把所有信息都塞进首屏。首屏吸引点击，第二屏负责持续阅读。

### 首屏布局建议

首屏建议采用：

1. 一个大主卡
2. 两个次卡

布局目标不是整齐平均，而是让主内容足够突出。

主卡：

1. 面积最大
2. 封面或视觉比重最高
3. 承担首页第一点击目标

次卡：

1. 支持补充内容多样性
2. 与主卡形成层级，而不是抢主卡焦点

### 第二屏布局建议

第二屏不建议三栏完全对称。

推荐的呈现方式：

1. 文章区：纵向列表
2. 播客区：纵向列表
3. 视频区：卡片网格

这样做的原因：

1. 文章更适合密度更高的阅读入口
2. 播客需要强调节目名、单集名和时长
3. 视频更依赖视觉封面，不适合与文章完全同构

## 各分区详细要求

### 文章区

文章区用列表展示最新文章。

每条建议包含：

1. 标题
2. 频道或作者
3. 发布时间
4. 可选的一小行摘要

文章区强调快速判断“值不值得读”。

### 播客区

播客区也采用列表，但信息重点不同于文章。

每条建议包含：

1. 单集标题
2. 节目或频道名称
3. 时长
4. 发布时间
5. 更明确的封面存在感

播客区应让用户一眼区分“这是音频内容”，而不是文章的轻度变体。

### 视频区

视频区使用卡片网格。

每条建议包含：

1. 视频封面
2. 标题
3. 频道或作者
4. 发布时间或时长

视频区需要比文章和播客更强视觉密度，否则用户会觉得它只是又一组文本列表。

## 创作入口

创作入口保留，但在首页中降级。

允许的入口位置：

1. Topbar 右侧
2. 首屏边角的次级按钮
3. 第二屏之前的小型辅助入口

不应把创作入口放回首页主视觉中心，也不应让首页默认跳入创作流程。

## 数据策略

首页第一阶段可以建立在现有公开最新内容能力之上。

数据策略如下：

1. 首屏精选内容：
   可以先由现有最新内容中选取，再按视觉或权重规则决定主次位。
2. 第二屏文章区：
   直接读取最新文章。
3. 第二屏播客区：
   直接读取最新播客单集。
4. 第二屏视频区：
   直接读取最新视频。

未来如果需要增强精选逻辑，可以在不改首页结构的前提下替换数据来源。

## 路由与行为要求

内容模块根路径应指向公开内容首页，而不是创作页。

行为要求：

1. 内容模块根路径默认进入公开内容首页。
2. 创作页继续保留登录保护。
3. 文章、播客、视频页继续作为各自独立的查看全部入口。

## 验证要点

需要验证以下行为：

1. 点击 topbar 中的内容模块，默认进入公开内容首页。
2. 未登录用户进入内容模块时，不会被直接送去登录页。
3. 首屏只显示精选内容，不显示解释性说明。
4. 第二屏包含文章、播客、视频三个分区。
5. 每个分区都能清晰进入对应的“查看全部”页面。
6. 创作入口依然存在，但不会盖过读者浏览入口。

## 结论

新的内容模块首页应是一个读者优先的内容门面，而不是创作者优先的工作台入口。

最终设计决策如下：

1. 首页第一优先用户是游客和读者。
2. 首页采用“首屏精选门面 + 第二屏类型分区”的两段式结构。
3. 首屏不放说明文字，只展示 1 个主精选和 2 个次精选。
4. 首页不再设置第三屏混合流。
5. 第二屏按文章、播客、视频三个分区组织最新内容。
6. 文章和播客使用列表，视频使用卡片网格。
7. 创作入口保留，但降级为次级动作。
8. 内容模块根路径默认进入公开内容首页，而不是创作页。
