# Atoman 全白双刻痕极简风设计系统规范

- Date: 2026-06-05
- Updated: 2026-07-12
- Status: Approved
- Scope: Atoman-Frontend 全局视觉令牌、应用框架、基础 UI 组件和页面布局

本规范是 Atoman-Frontend 的唯一全局视觉基准。模块专项文档负责业务流程、页面能力和信息架构；若其中的颜色、排版、圆角、阴影、组件或交互反馈规则与本规范冲突，以本规范为准。单独获批的模块设计可以调整信息架构，单纯视觉重构不得顺带修改导航名称、顺序或页面结构。iOS 使用独立设计体系。

## 1. 设计目标

Atoman-Frontend 采用 **全白双刻痕极简风（Pure-White Double-Notch Minimal System）**。系统以纯白表面、冷灰层次、轻量边界和明确动作层级为基础，移除旧有的新粗野主义粗边框、重字重、硬阴影及装饰性纸张堆叠。

整体界面应保持安静、清楚和可持续阅读。视觉层级主要由留白、排版、浅色背景和少量边界建立，而不是依赖阴影、厚重描边或大面积装饰色。

## 2. 核心视觉语言

### 2.1 纯白表面

- 应用背景、Topbar、Sidebar、卡片、下拉面板、Modal 和 Sheet 的基础表面使用 `#ffffff`。
- 可交互区域的弱背景使用冷灰色，不使用暖黄纸色或模拟旧纸张纹理。
- 页面层次优先通过间距、排版和局部浅灰背景表达。

### 2.2 轻量边界

- 常规边线使用 `#cbd5e1`。
- 柔和分隔线使用 `#e2e8f0` 或 `#f1f5f9`。
- 不使用 2px 粗黑边框。
- Topbar、Sidebar 和页面布局不得依赖贯穿整页的水平或垂直长分割线。

### 2.3 无阴影

以下全局阴影令牌统一为 `none`：

```css
--a-shadow-button: none;
--a-shadow-dropdown: none;
--a-shadow-modal: none;
```

基础组件不得通过硬投影、纸张投影或悬浮位移表达层级。交互反馈使用背景色、文字色、边界色和最多 `1px` 的按下位移。

### 2.4 小圆角

- 按钮、输入容器、下拉面板、卡片、媒体封面、Textarea、Modal、Sheet 和头像统一使用 `4px` 圆角。
- Radio 保持圆形。
- 状态指示点保持圆形。
- 不使用胶囊形按钮或大面积圆角卡片；只有语义上需要连续选择轨道的 Segmented Control 可以保留紧凑的滑块结构。

### 2.5 克制字重

```css
--a-font-weight-black: 500;
```

- 页面标题与组件标题默认使用 `500` 字重。
- 页面标题、组件标题和卡片标题不使用 `700` 至 `950` 字重制造层级。
- 层级通过字号、间距、位置和颜色建立。

## 3. 色彩系统

### 3.1 中性色

```css
--a-color-paper: #ffffff;
--a-color-ink: #0f172a;
--a-color-ink-muted: #334155;
--a-color-line: #cbd5e1;
--a-color-line-soft: #f1f5f9;
--a-color-paper-wash: #f1f5f9;
--a-color-paper-soft: #f8fafc;
```

### 3.2 Primary 明晰蓝

Primary 动作统一使用用户确认的 **明晰蓝（Clear Blue）**：

```css
--a-color-primary: #2563eb;
--a-color-primary-hover: #1d4ed8;
--a-color-primary-pressed: #1e40af;
--a-color-primary-contrast: #ffffff;
```

使用规则：

- 每个独立操作组原则上只出现一个 Primary 按钮。
- Primary 用于发布、提交、确认、创建、继续等当前流程的主要动作。
- Hover 使用 `#1d4ed8`。
- Pressed 使用 `#1e40af`，并允许 `translateY(1px)` 的轻微位移。
- Focus 必须保留清晰的键盘可见轮廓，建议使用 `2px solid #2563eb` 并设置外偏移。
- Disabled 不创建新的灰色变体；保留当前颜色并降低透明度，同时移除交互指针。
- Loading 保持 Primary 蓝色语义，同时显示进度状态并禁止重复点击。
- 蓝色仅用于动作与必要的交互焦点，不扩散为大面积页面背景或装饰色。

### 3.3 Secondary、Ghost 与 Danger

- **Secondary**：`#f1f5f9` 背景、`#0f172a` 文字、无可见边框；Hover 加深到 `#e2e8f0`。
- **Ghost**：透明背景、`#0f172a` 文字；Hover 使用 `#f8fafc`。
- **Danger**：使用红色 `#dc2626`，用于删除、驳回等破坏性动作；按钮使用白色背景、危险色文字与浅危险色边界。
- **Warning**：使用橙色 `#ea580c`，用于警告、待处理和可恢复异常。
- **Success**：使用绿色 `#0d9488`，用于成功状态，也可用于音乐播放进度等明确的进度语义。
- “取消”默认是普通次级动作，不因文案本身自动使用 Danger。
- 不得使用 Primary 蓝色表达删除等高风险动作，也不得让模块点睛色替代全局 Primary。

## 4. 双刻痕应用框架

双刻痕是应用框架的识别元素，不应复制到每张卡片或每个组件中。

### 4.1 顶部居中横刻痕

Topbar 底部中央放置一段深板岩色横线：

```css
width: 20px;
height: 1px;
background: #334155;
left: 50%;
transform: translateX(-50%);
```

### 4.2 侧栏交界竖刻痕

Sidebar 与主内容交界位置放置一段深板岩色竖线：

```css
width: 1px;
height: 16px;
background: #334155;
top: 50%;
transform: translateY(-50%);
```

桌面端基准位置为距左侧 `180px` 的框架分割点；实际实现应依附布局变量或侧栏宽度，而不是在多个页面中重复硬编码。

### 4.3 响应式规则

- 移动端隐藏或随布局重新定位侧栏交界竖刻痕，不能让刻痕悬浮在无实际交界的位置。
- 顶部横刻痕可以保留，但不得遮挡标题、返回操作或安全区。
- 双刻痕只负责建立框架识别，不承担导航、返回或拖动交互。

## 5. 基础组件规范

### 5.1 `PButton.vue`

- 圆角：`4px`。
- 字重：`500`。
- 动效时长：`0.15s`。
- Primary：`#2563eb` 背景、白色文字、同色边界。
- Primary Hover：`#1d4ed8`。
- Secondary：`#f1f5f9` 背景、`#0f172a` 文字、透明边界。
- Ghost 与 Danger 遵循第 3.3 节。
- 支持 `sm`、`md`、`lg` 三种尺寸以及 block、disabled、loading、dot 状态。
- `dot` 仅表达真实状态，不作为装饰。

### 5.2 `PInput.vue`

- 使用下划线输入样式，只保留 `1px solid #cbd5e1` 底边。
- 背景透明或纯白。
- Focus 底边切换为 `#2563eb`，并保持键盘焦点可见。
- 不通过完整边框、阴影或大圆角表达输入状态。

### 5.3 `PSelect.vue`

- 触发区域与 `PInput` 使用一致的底线样式。
- 选项面板使用 `1px solid #e2e8f0`、`4px` 圆角和纯白背景。
- 面板无阴影。
- Option Hover 使用 `#f1f5f9`。
- 当前选项与键盘焦点可使用 Primary 明晰蓝，但必须保证文字对比度。

### 5.4 `PTextarea.vue`

- 使用 `2rem` 行高的淡色横向规则线，形成克制的稿纸提示。
- 默认只保留底边，不使用完整重边框。
- Focus 使用 Primary 蓝色提示当前输入状态。
- 若实现光标所在行高亮，必须由真实光标位置驱动，并保证滚动、换行和字体缩放后仍准确；无法可靠实现时，保留静态规则线，不提供错误的动态高亮。

### 5.5 `PEntry.vue`

- 列表项默认无卡片边框和阴影。
- 使用间距与排版形成条目分组。
- Hover 背景切换到 `#f8fafc`。
- Hover 可显示细竖向左指示线；指示线不得导致内容横向跳动。

### 5.6 `PSegmentedControl.vue`

- 容器背景使用 `#f1f5f9`。
- Active 项使用 `#ffffff` 背景和 `4px` 圆角。
- 不使用阴影模拟浮起。
- Active 文字或焦点提示可使用 Primary 蓝色，但不能仅依赖颜色表达选中状态。

### 5.7 `PBadge.vue`

- 默认不使用胶囊背景。
- 文本前放置 `6px` 状态圆点。
- 推荐状态色：成功绿色 `#0d9488`、警告橙色 `#ea580c`、危险红色 `#dc2626`。
- 状态必须同时有文字标签，不能只依赖圆点颜色。

### 5.8 `PChoiceField.vue`

- Checkbox 使用 `4px` 圆角。
- Radio 保持圆形。
- 选中态优先使用 `#2563eb`，并提供勾选或内点形状。
- 键盘焦点必须可见。

### 5.9 `PDropdown.vue`

- 使用 `1px solid #e2e8f0` 边界、`4px` 圆角和纯白背景。
- 无阴影。
- Hover 与选中态遵循 Select 的冷灰与 Primary 蓝规则。

### 5.10 `PClip.vue` 与 `PDiscussionFAB.vue`

- `PClip` 使用 `4px` 圆角与 `1px solid #e2e8f0` 边界。
- `PDiscussionFAB` 使用 `4px` 圆角方形，不使用圆形悬浮按钮造型。
- 两者均保持扁平，不使用投影。

### 5.11 `PModal.vue` 与 `PSheet.vue`

- 内容表面使用纯白背景、`1px solid #e2e8f0` 边界和 `4px` 圆角。
- 不使用卡片阴影。
- Overlay 使用 `rgba(15, 23, 42, 0.08)`。
- 可使用 `backdrop-filter: blur(8px)`，但必须提供不支持该属性时仍可读的纯色 Overlay。
- Modal 是单层聚焦容器。Sheet 支持按语义层级动态堆叠：进入新实体或独立任务时 `push` 新层，Tab、筛选、排序、分页和同一表单内的步骤不增加层级。
- Sheet 堆栈不设置固定上限，至少支持“基础页面 → 艺术家详情 → 专辑详情 → 修订 → 历史”等连续上下文。
- 桌面端可以露出下层 Sheet 的边缘并允许返回指定层；移动端保留完整逻辑堆栈，但只完整显示当前层。
- 浏览器返回与 Sheet 返回只退出当前顶层。直接打开实体链接时恢复可确定的实体上下文，不恢复临时筛选状态。
- 全局播放器等持续性组件位于 Sheet 堆栈之外，切换层级不得中断播放。

### 5.12 `PEmpty.vue`

- 不使用完整盒状边框。
- 仅保留 `1px solid #f1f5f9` 的顶部和底部分隔线。
- 文案保持简洁，只说明当前状态和可执行动作。

### 5.13 `PLink.vue`

- 默认不显示下划线，Hover 时显示。
- 仅对明确跳往外部页面或新上下文的链接追加 `↗`；站内普通导航不得统一追加外链符号。
- Hover 时箭头可轻微向右上移动。

### 5.14 `PAvatar.vue`

- 使用 `4px` 方形圆角，不使用圆形头像。
- 图片裁切使用 `object-fit: cover`。
- 缺省头像保持可辨识，不加入装饰阴影。

### 5.15 `PToast.vue`

- 使用纯白背景、`1px solid #e2e8f0` 边界和 `4px` 圆角。
- 无阴影。
- 使用 `6px` 状态圆点与简洁文字共同表达结果。
- Toast 不使用 Primary 蓝色替代成功、警告或错误语义色。

### 5.16 功能图标

- 功能图标统一使用 Lucide，不使用 Emoji 或各模块自绘图标表达同一动作。
- 普通命令使用图标加文字。关闭、返回等熟悉工具可以只显示图标，但必须提供 Tooltip 和 `aria-label`。
- 陌生图标不得脱离文字单独出现；装饰性图标不进入操作区。

## 6. 页面与布局规则

- 风格更新不得改变 Sidebar 的名称、路由、顺序或分组；单独获批的模块功能设计可以调整信息架构。
- 清理 `BlogLayout.vue`、`FeedLayout.vue` 等布局中的贯穿式竖向分隔线和非必要背景偏移。
- 页面标题、卡片标题和导航强调字重收敛至 `500`。
- 页面结构主要依靠一致的宽度、留白与节奏，不使用重复边框包裹每个区块。
- 用户可见文案遵循项目规则：标题只写动作名或内容名，说明仅保留完成当前任务所需信息。
- 不添加装饰性 Emoji 或无功能图标。
- 视频卡片的播放量、时长等统计信息始终显示。
- “稍后看”等次级操作在桌面端于 Hover 或 Focus 时显示，在触摸设备上始终显示，并始终保持键盘可访问。

## 7. 交互与无障碍

- 所有交互组件必须提供可见的 `:focus-visible` 状态。
- Primary 蓝色文字或背景组合必须达到 WCAG AA 对比度要求；白字搭配 `#2563eb` 可用于常规尺寸按钮。
- Disabled 状态必须同时阻止点击并设置正确的 `disabled` 或 `aria-disabled`。
- Loading 状态必须防止重复提交，并用文字说明当前动作。
- Hover 不能作为唯一反馈；键盘与触摸设备必须获得等价状态。
- `prefers-reduced-motion: reduce` 下关闭非必要位移和动画。
- 打开 Sheet 后焦点进入新层；关闭后焦点返回触发元素；`Esc` 只关闭当前顶层。
- 表单提交失败时保留用户输入，在当前层提供可执行的修正提示，不自动关闭或退层。
- 实体加载失败时保留返回路径并提供重试，不清空整个 Sheet 堆栈。

## 8. 实施范围

### 8.1 全局令牌

更新 `src/style.css`：

- 圆角基线设为 `4px`。
- 标题重字重令牌设为 `500`。
- 引入 Primary 明晰蓝及 Hover、Pressed、Contrast 令牌。
- 移除按钮、下拉面板和 Modal 阴影。
- 收敛纸张、边线与冷灰背景令牌。
- 建立双刻痕使用的框架级尺寸与颜色变量，避免页面重复硬编码。

### 8.2 基础组件

按第 5 节更新 `src/components/ui/` 下对应组件。组件公共 API 应保持稳定，除非现有 API 无法表达规范要求。

### 8.3 Sheet 堆栈

- 在现有 `PSheet.vue`、`PSheetTab.vue` 和音乐抽屉基础上增加通用堆栈控制器。
- 通用控制器只管理 `push`、`replaceTop`、`pop`、`popTo`、层级顺序和焦点恢复，不包含模块业务逻辑。
- 音乐模块现有 `useMusicDrawers.ts` 首先迁移；其他模块按实际需要逐步接入，不一次性重写全部页面。
- 实体组件继续自行负责数据加载、提交和错误状态。

### 8.4 布局

- 在共享应用框架中实现双刻痕，而不是在每个模块布局重复实现。
- 移除模块布局中冗余的长边界与旧纸张阴影。
- 检查桌面、平板与移动端的刻痕位置和组件密度。

## 9. 验收标准

### 9.1 静态验证

```bash
bun run type-check
bun run test:unit
bun run build
```

所有命令必须成功。

### 9.2 组件验证

至少验证：

- `PButton` 四种 variant、三种 size、Hover、Pressed、Focus、Disabled、Loading 与 block 状态。
- `PInput`、`PSelect`、`PTextarea` 的默认、Focus、错误与 Disabled 状态。
- `PChoiceField` 的 Checkbox、Radio 和键盘操作。
- `PDropdown`、`PModal`、`PSheet`、`PToast` 的开关、焦点与 Overlay。
- `PBadge` 的状态圆点与文字标签。
- `PLink` 的站内链接与外部链接差异。
- Sheet 堆栈的 `push`、`replaceTop`、`pop`、`popTo`、焦点恢复、错误保留和移动端单层显示。

### 9.3 页面验证

在真实浏览器中至少检查：

- 桌面端 Topbar 居中横刻痕与 Sidebar 交界竖刻痕位置正确。
- 移动端不存在悬空的 Sidebar 交界刻痕。
- Blog、Feed 及一个包含表单的页面没有残留粗边框、硬阴影或 `700` 至 `950` 标题字重。
- Primary 明晰蓝在白色、冷灰背景和 Modal 中均保持清晰一致。
- 页面不存在因 Hover 指示线或 Pressed 位移造成的布局跳动。
- 键盘导航时焦点顺序与焦点轮廓清楚。
- 动态 Sheet 在桌面端能返回任意下层，在移动端能逐层退出，浏览器返回只移除顶层。
- 视频统计信息始终可见，次级操作在桌面 Hover/Focus 与触摸设备上均可访问。
- 全局播放器在 Sheet 层级切换期间不中断。

## 10. 非目标

本次风格系统更新不包含：

- 在纯视觉重构中修改 Sidebar 的路由、名称、顺序或分组；独立获批的模块功能设计不受此项限制。
- 重构业务 API、状态管理或页面数据流。
- 引入新的大型 UI 组件库。
- 恢复模块前缀路由。
- 重写模块业务数据流以适配 Sheet；业务组件通过通用堆栈接口接入即可。
- 为了展示 Primary 蓝色而给大面积页面背景染色。
