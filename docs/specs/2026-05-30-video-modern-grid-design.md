# Specification: 视频 (Video) 模块现代极简重构

- **Status**: Superseded in part
- **Date**: 2026-05-30
- **Author**: Gemini CLI

> 当前规则：本文件的侧边栏结构已由[视频上传到观看闭环设计](../superpowers/specs/2026-07-09-video-upload-watch-loop-design.md)取代，采用“探索、订阅、收藏、创作”。圆角、头像、边界、阴影、字重、图标和交互反馈由[全局 UI 规范](./2026-06-05-flat-paper-ui-design.md)取代：所有媒体封面与头像使用 `4px` 圆角，不使用硬阴影或粗黑边，标题字重使用 `500`。播放量、时长等统计信息始终显示；“稍后看”等次级操作在桌面 Hover/Focus 时显示，在触摸设备上始终显示，并保持键盘可访问。16:9 网格、两行标题截断与内容布局仍然有效。

## 1. 背景与目标 (Background & Motivation)
为了进一步提升全站视觉一致性，同时兼顾视频内容的高效发现，我们将重构视频模块。本次重构抛弃了硬框线的束缚，转向一种更开阔、更具现代感的 YouTube 式网格布局，同时在排版和交互细节上保留“法堂”模块的精致触感。

## 2. 布局与导航 (Layout & Navigation)

### 3.1 侧边栏对齐 (Sidebar Structure)
当前侧边栏结构依次为：
1. **01/ 探索 (Explore)**：对应路由 `/`。展示全站精选与最新视频。
2. **02/ 订阅 (Subscriptions)**：对应路由 `/subscriptions`。展示用户关注的频道动态。
3. **03/ 收藏 (Favorites)**：展示收藏的视频与合集。
4. **04/ 创作 (Creator)**：进入上传与创作管理。

### 3.2 网格系统 (Grid Layout)
- **4 列布局**：在大屏幕下强制采用 4 列网格（`repeat(4, 1fr)`）。
- **去框化设计**：移除外层卡片边框，利用 1.5rem - 2rem 的大间距（Gap）自然区分内容。

## 3. 视频卡片 (VideoCard) 核心规范

### 3.1 封面区 (Visual Overlay)
- **形状**：16:9 比例，`4px` 圆角矩形。
- **播放数据 (左下角)**：`▶ {播放量}`，始终显示。
- **时长标识 (右下角)**：`{MM:SS}`，始终显示。
- **稍后观看 (右上角)**：
  - 样式：白色背景、`1px` 浅色边界、`4px` 圆角，无阴影。
  - 交互：桌面端在 Hover 或 Focus 时显示，触摸设备始终显示，并保持键盘可访问。

### 3.2 信息区 (Metadata & Typography)
- **头像展示**：保留 36px 频道/作者头像，使用 `4px` 方形圆角。
- **标题样式**：
  - 使用 `500` 字重，不使用 `700` 至 `950` 重字重。
  - **交互**：悬停或聚焦时显示下划线。
  - 限制为 2 行，超出截断。
- **辅助行**：使用 `SFMono` 字体显示 `《频道名》 · {日期}`，颜色使用 `var(--a-color-muted-soft)`。

## 4. 管理页与订阅 (Interaction)
- **两段式管理**：顶部选择合集，下方展示并管理该合集内的视频流。
- **订阅页**：纯净的关注内容发现流，布局与探索页一致。

## 5. 验证准则 (Verification)
- 验证在大屏幕下是否严格呈现 4 列布局。
- 验证统计信息始终显示，封面右上角“稍后看”在桌面 Hover/Focus 与触摸设备上均可访问。
- 验证标题 Hover/Focus 时的下划线反馈。
- 验证新版四项侧边栏的数字编号及激活态。
