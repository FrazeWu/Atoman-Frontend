# 音乐歌词逐行编辑器设计

## 背景

歌词 Stage 1 已提供歌曲级 Wiki 歌词、纯文本与 LRC、双语展示、版本恢复、注释、投票和锚点冲突处理。当前编辑器仍使用原文和翻译两个大文本框，不适合逐行对齐、时间校验和 LRC 文件交换。

本阶段实现 Stage 2 的第一部分：建立统一逐行草稿模型和可用的表格编辑工作流，为后续播放辅助打轴提供稳定基础。

## 目标

1. 以统一行模型编辑原文、翻译和可选时间。
2. 支持逐行增加、删除、上移和下移。
3. 支持原文 LRC 与可选翻译 LRC 的导入预览，确认后整体替换草稿。
4. 分别导出原文 LRC 和翻译 LRC。
5. 校验空原文、时间倒序和重复时间戳。
6. 复用现有歌词保存、版本和注释冲突协议。
7. 在桌面和移动端提供稳定、无重叠的编辑界面。

## 非目标

- 播放辅助打轴。
- 拖拽调整时间。
- 视觉注释重新绑定。
- 单词级时间戳。
- 新增或修改后端歌词 API。
- 自动猜测单文件混合双语 LRC。

## 数据模型

前端使用统一草稿行：

```ts
export interface MusicLyricDraftRow {
  id: string
  timeMs: number | null
  original: string
  translation: string
}
```

`id` 仅用于本地渲染稳定性，不发送到后端。纯文本和 LRC 共用该模型：纯文本忽略 `timeMs`，LRC 使用 `timeMs`。

## 模块边界

### `musicLyricsDraft.ts`

纯函数模块，负责：

- 将现有 `content + translation + format` 解析为草稿行。
- 将草稿行序列化为现有保存 API 的 `content + translation`。
- 解析原文 LRC 和可选翻译 LRC。
- 按时间戳及同时间戳出现顺序配对翻译。
- 格式化和解析 `mm:ss.xx` 时间输入。
- 校验、按时间排序和生成两个 LRC 导出文本。

该模块不访问浏览器 API，不管理 Vue 状态。

### `MusicLyricsRowEditor.vue`

负责行级编辑和操作。桌面使用表格网格，列为序号、时间、原文、翻译和操作；纯文本模式隐藏时间列。移动端每行切换为时间、原文、翻译和操作的纵向布局。

第一版使用上移、下移按钮，不实现拖拽。

### `MusicLyricsImportPreview.vue`

负责展示导入结果、错误和提醒。解析成功且不存在阻断错误时允许确认；确认前不修改当前草稿。

### `MusicLyricEditorDrawer.vue`

负责格式切换、导入、导出、添加行、校验摘要和保存。它向父组件输出与现有接口相同的 payload。

### `MusicLyricsPanel.vue`

继续负责调用保存 API、版本状态和 409 注释冲突处理。新编辑器不复制这些逻辑。

## 解析与序列化

### 纯文本

- 原文按物理行生成草稿行。
- 翻译按相同行号配对。
- 缺失翻译为空字符串。
- 保留中间空翻译位置。
- 保存时按当前行顺序分别拼接原文和翻译。

### LRC

- 原文 LRC 是行集合的主表。
- 翻译 LRC 可选。
- 接受 `[mm:ss.xx]` 和 `[mm:ss.xxx]` 时间标签；同一物理行包含多个时间标签时按标签顺序展开为多行。
- 标题、作者等非时间元数据不进入歌词行；不支持的控制标签在预览中明确提示。
- 翻译按“时间戳 + 同时间戳出现顺序”与原文配对，支持合法的重复时间戳。
- 原文缺少翻译允许导入。
- 翻译中存在无法匹配的时间戳或多余同时间戳行时阻止确认，避免静默丢失。
- 导入确认后格式切换为 LRC，并整体替换当前草稿。

保存继续调用现有 `PUT /api/v1/music/songs/:songId/lyrics`。LRC 原文和翻译分别序列化，后端版本、稳定行 ID 和注释锚点逻辑保持不变。

## 导入与导出

导入入口接收：

1. 必选原文 `.lrc`。
2. 可选翻译 `.lrc`。

文件读取或解析错误显示文件名和具体行号。取消预览保持原草稿；确认预览才替换草稿，仍需点击保存才写入服务器。

导出生成：

- `<歌名>.lrc`
- `<歌名>-translation.lrc`

翻译文件只包含有翻译内容的行。导出按钮仅在 LRC 模式且时间校验通过时可用。

## 校验规则

### 阻止保存

- 任一草稿行的原文为空。
- LRC 行缺少时间。
- 时间戳倒序。
- 时间输入格式无效。
- 导入的翻译行无法匹配原文。

### 仅提醒

- 重复时间戳。

时间倒序时提供“一键按时间排序”。排序保持相同时间戳行之间的原相对顺序。

## 交互

工具栏包含格式切换、导入 LRC、导出原文、导出翻译和添加一行。行操作使用上移、下移和删除图标按钮，并提供 tooltip 和明确的无障碍名称。

编辑器打开时从当前歌词重新建立草稿；取消关闭不保存。保存中禁用所有会改变草稿或再次提交的操作。格式切换只改变当前编辑方式：在本次打开期间从 LRC 切到纯文本不会立即清除本地时间值，但以纯文本保存后服务器不再保留时间。

## 错误处理

- 导入错误留在预览层，不修改草稿。
- 本地校验错误定位到具体行。
- 导出失败显示简短用户态错误。
- 服务端保存错误和 409 冲突继续由 `MusicLyricsPanel.vue` 处理。
- 切歌或关闭歌词面板时沿用现有请求代际保护，不允许旧草稿写入新歌曲。

## 测试

### 单元测试

- 纯文本解析和序列化往返。
- LRC 解析和序列化往返。
- 重复时间戳按出现顺序配对翻译。
- 空翻译、无法匹配翻译和解析行号错误。
- 时间格式化、倒序校验、稳定排序和重复提醒。

### 组件测试

- 增加、删除、上移和下移。
- 纯文本与 LRC 列切换。
- 导入取消不修改草稿，确认整体替换。
- 校验阻止保存，重复时间仅提醒。
- 分别导出原文和翻译。
- 保存 payload 继续保留物理行结构。

### 回归与浏览器测试

- 现有歌词保存、版本恢复和 409 草稿保留测试继续通过。
- Chromium 验证真实文件导入和下载。
- 验证桌面表格和 390px 移动布局无重叠。

## 验收标准

1. 用户可以只通过逐行界面完成纯文本或 LRC 双语歌词编辑。
2. 双文件 LRC 导入不会静默丢失或错配翻译。
3. 空原文和倒序时间不能保存，重复时间可以保存但有提醒。
4. 导出文件可再次导入并保持行、时间和翻译语义。
5. 现有版本、注释冲突和请求竞态保护不回归。
